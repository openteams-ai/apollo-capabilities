Welcome

Build & run AI-ready capabilities

Apollo capabilities are self-contained, reproducible environments that expose tools, models, and apps with a single command.

🧩 What is a capability?

A capability is a Nebi-managed pixi project that packages everything needed to run an AI tool, model, or application. Each capability declares:

  • Its dependencies — pinned conda/PyPI packages
  • A launch task — the single command that starts the app or service
  • Metadata — name, description, target environments, and how it should run (app, browser, or background)
Option 1 — Apollo Desktop App
1

Open the Apollo Desktop App

Launch Apollo Desktop on your machine. All available capabilities are listed in the capabilities browser.

2

Select a capability

Click on any capability card to open its detail view, where you can read its description and requirements.

3

Click Install

Apollo resolves and installs all dependencies into an isolated environment, then launches the capability automatically — no terminal needed. It opens in the correct context: app window, browser tab, or background service.

Option 2 — Run locally with Nebi
1

Install Nebi

Nebi builds on pixi to add environment versioning, sharing, and team governance. Install pixi first, then install Nebi.

curl -fsSL https://pixi.sh/install.sh | sh
pixi global install nebi
2

Import the capability

Pull the capability directly from the OpenTeams registry into a local directory.

nebi import quay.io/openteams_capabilities/hello-world:latest -o ./hello-world
3

Launch the capability

Navigate into the imported directory and run the launch task with pixi.

cd hello-world
pixi run launch
Option 3 — Run locally with pixi
1

Install pixi

Pixi is the package manager that powers every capability. Install it once on your machine.

curl -fsSL https://pixi.sh/install.sh | sh
2

Clone the repository

Clone the Apollo capabilities repository to get all capabilities locally.

git clone https://github.com/openteams-ai/apollo-capabilities.git
cd apollo-capabilities
3

Navigate to a capability directory

Each capability lives in its own folder inside capabilities/.

cd capabilities/hello-world
4

Run the launch task

Pixi installs dependencies into an isolated environment on first run, then executes the launch task.

pixi run launch
Creating a new capability
1

Copy an existing capability as a template

The hello-world capability is the simplest starting point. Copy it into a new folder with your capability's name.

cp -r capabilities/hello-world capabilities/my-capability
2

Update the manifest

Edit pixi.toml to set the name, description, dependencies, and launch task for your new capability.

# Edit capabilities/my-capability/pixi.toml
3

Run it

Pixi installs your dependencies and launches the capability.

cd capabilities/my-capability
pixi run launch
